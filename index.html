Must Master HTML Tags

    google Doc to all here Guide HTML Layout/Techniques
  • html
    • The start and end of the html doc, its acts as a wrapper for all other elements
  • head
    • Cotains the main information about the doc includinding title, style and script
  • title
    • The title of the doc, shown in the browser, history or bookmark. Lives in the head tag
  • style
    • Contain style info for some of the doc if not all, it contains css, to apply inside tag: style =..., Lives in the head
  • body
    • wrapes the contents of the doc
  • script
    • Allows you to run Js or Json or Webjl scripts on your html webpage. When the browser see a script while parsing, it pauses, downloads and excutes the script. Attributes like async and defer control when and how the script it excuted in relation to parsing.
  • h1, h2, h3
    • Heading, goes from 1 to 6
  • p
    • Paragraphs, text or any structural grouping of related content, such as images or form fields.
  • a
    • Anchor, creates hyperlink
  • img
    • Embeds an image into the doc, always specify alt, with and hight
  • ul
    • Unordered list
  • header
    • for introductory content, logo, search bar and More
  • footer
    • for info like copyriht, author and more
  • section
    • for section of a doc and should always have headers, general content grouping
  • nav
    • for navigation links
  • article
    • self contained section within the doc, shoud be sence on its own, be independently distributable or reusable, often used for blog posts, new articles, product cards and forum posts can be nested within section or stand alone
  • main
    • for main content of doc, 1 per page, not duplicated on other pages, CANNOT be nested in elements like article, aside, footer, header, or nav.
  • li
    • list of items, could be in an ul or ol or menu
  • div
    • Used to group related content, block lvl emelent
  • span
    • Just like div but inline lvl element
  • table
    • For table data, rows and colums
  • tr
    • A row in a table
  • td
    • Defines data of a cell table
  • form
    • Used to send informtion from front end to back end
  • input
    • Used to accept data from users
  • textarea
    • For really big text input
  • select
    • A control to provide a menu of options
